## Process notes (operational discipline)

1. **Every label flip MUST be followed by a `gh pr view N --json labels` verification query.** Never trust the command output alone — `gh pr edit` silent failures are a known artifact (TD-004).
2. **Every ADR / design doc / tech-debt edit MUST be followed by a `git status` check** to confirm the file is in the working tree (or committed, depending on intent). Multi-agent branch contention (TD-003) can cause edits to land on the wrong branch or be lost.
3. **`gh pr edit` with multiple --add-label + --remove-label in one call is unreliable.** Use separate calls for atomicity.
4. **Cross-agent label state is not stable.** After a label flip, another agent's automation or workflow may re-add labels. Re-verify before assuming the flip stuck.
